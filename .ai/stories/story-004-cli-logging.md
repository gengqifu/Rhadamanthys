# 用户故事：CLI、日志与预检
- 状态：Draft
- 故事点：3 SP

## 背景
工具需离线运行，提供进度/错误日志、预检依赖，并支持可配置输出。

## 需求
- CLI：`python cli.py scan <project_path> --out report.xlsx [--rules ...] [--verbose] [--debug] [--log-interval-ms N] [--format excel,json,csv]`
- 日志：默认中文；默认日志间隔 1s，可调至 30ms（debug），提示 I/O 开销；按模块输出阶段性汇总。
- 预检：启动前检查 Python 版本（2.7.18）、依赖包、libclang 路径与输入路径，失败给出中文错误与退出码。
- 退出码约定：0 成功；1 参数/路径错误；2 依赖缺失；3 扫描中断/异常。
- 配置忽略目录/文件大小上限，并可覆盖。

## 验收标准
- CLI 能解析上述参数并按配置调整输出与日志频率。
- 预检失败时输出清晰错误与正确退出码。
- 长任务可见持续进度日志，阶段性汇总清晰。
